# AGENTS.md（最小实现版）

目标：在不改变现有功能（模式切换、倒计时、环形进度、开始/暂停/重置、主题切换、结束提醒）的前提下，用最少步骤完成开发与交付。

## 1. 最小代码约定

- 只维护 3 个核心文件：`index.html`、`style.css`、`script.js`。
- 注释可少但必须中文，只解释关键决策。
- 不做额外抽象：能在当前文件清晰实现，就不拆新模块。
- 不引入新框架、不重构目录（除非需求明确升级桌面端）。

## 2. 最小测试流程（固定 3 步）

1. 语法检查：`node --check script.js`
2. 页面手测：
- 模式切换后时间正确
- 开始/暂停/重置行为正确
- 倒计时结束有提示音/通知（权限允许时）
- 主题切换正常
3. 提交前确认：页面无白屏、无明显布局错位

## 3. 最小目录约定

- `index.html`：结构与可访问性属性
- `style.css`：样式与动效
- `script.js`：状态与计时逻辑
- `request.md`：需求输入
- `AGENTS.md`：协作规范

说明：`docs/`、`tests/`、`assets/` 暂不强制创建，需要时再加。

## 4. 最小提交流程

1. 改动代码
2. 跑完“最小测试流程”
3. 提交：
- `feat:` 新功能
- `fix:` 修复
- `style:` 样式调整
- `docs:` 文档
4. 推送：`git push`

回滚原则：优先用 `git revert` 生成回滚提交，不改写公共历史。

## 5. 取舍说明

- 取舍 1：不引入自动化测试框架
  - 保留：功能正确性验证
  - 放弃：更高自动化覆盖率
  - 原因：当前是小型静态项目，人工回归成本更低。

- 取舍 2：不拆模块、不建额外目录
  - 保留：功能完整与可维护的基本结构
  - 放弃：中大型项目的扩展弹性
  - 原因：当前代码量小，减少文件能降低沟通和修改成本。

- 取舍 3：提交流程压缩为“改动-检查-提交-推送”
  - 保留：可追溯与可回滚
  - 放弃：复杂分支治理
  - 原因：当前协作规模小，简流程更高效。
